A PowerShell telepítése

A Windows PowerShell 2.0 a Windows 7 és a Windows Server 2008 R2 része, de a Windows Management Framework részeként (letölthető a http://support.microsoft.com/default.aspx/kb/968929 helyről) telepíthető Windows XP SP3, Windows Server 2003 SP2 és ezeknél újabb operációs rendszerre. A telepítés előfeltétele a .NET Framework 2.0 megléte a gépen. Mind a Windows PowerShell, mind a .NET Framework letölthető a Microsoft weboldaláról.

A Windows PowerShell 3.0 része a Windows 8-nak és Windows Server 2012-nek, a Windows PowerShell 4.0 pedig a Windows 8.1-nek és a Windows Server 2012R2-nek. A Windows PowerShell 3 telepíthető Windows 7/Windows Server 2008 vagy későbbi operációs rendszerekre, az előfeltételek között a .NET Framewok 4.0 szerepel. A Windows PowerShell 4.0 szintén telepíthető, előfeltétel a .NET 4.5 megléte.

A Windows PowerShell 5.x része a Windows 10-nek és része a Windows Server 2016-nak, valamint telepíthető a Windows 7 vagy újabb operációs rendszerekre, a .NET 4.5 itt is az előfeltétel.

A Windows PowerShellt a Windows 7-ben és generációjában a Start menü All Programs/Accessories/Windows PowerShell csoportjában található Windows PowerShell parancsikon segítségével, vagy közvetlenül a powershell.exe meghívásával indíthatjuk. A 64 bites Windows-okon két PowerShell ikonunk is lesz, egy 64 bites és egy 32 bites parancssori környezet indítására:

1 . ábra PowerShell programcsoport ikonjai Windows 7-ben (64 bites)

Találhatunk itt még egy (esetleg kettő) PowerShell ISE parancsikont is, erről majd kicsit később.

A Windows 10 operációs rendszerben és generációjában (Windows Server 2016) kicsit másképp találjuk meg az összes Windows PowerShellel kapcsolatos ikont:

2 . ábra Windows PowerShell ikonok a Windows 10 Start menüjében

Itt érdemesebb inkább a menü megnyitása után rögtön el kezdeni gépelni, hogy Windows PowerShell, és a vonatkozó ikonok megjelennek a keresési találatlista tetején.

Ha merészek vagyunk, akkor letölthetjük a PowerShell 7-et is immár 2020-tól, ami olyan, mint egy bármilyen alkalmazás, azaz nem a Windows könyvtárba települ, hanem a Program Files alá:

3 . ábra PowerShell 7.0 - a Program Files alatt

Fontos, hogy immár nem „Windows PowerShell”-ről, hanem csak „PowerShell”-ről beszélünk. A megkülönböztetés mögött az van, hogy immáron 5.2-es verzió óta a PowerShell nyílt forráskódú projektté vált, fent van a GitHub-on (https://github.com/PowerShell/PowerShell) és nem csak a Windows platformon, hanem Linuxon, és OSX-en is futtatható.

A start menüben is az alkalmazások között látjuk:

4 . ábra PWSH 7 a Start menüben

Kezdőknek nem ajánlom a PowerShell 7-et, mivel nincsen hozzá grafikus szkriptszerkesztő, így a megismerkedéshez inkább használjuk a Windows-ba beépített Windows PowerShell 5.x-et!
